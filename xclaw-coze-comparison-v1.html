XCLAW POSITIONING

XClaw 走的是第三条路

既不抄 CC/Cursor/Codex，也不简单照搬扣子编程——教学场景需要再下沉一层

XClaw 的差异化路线

不让教师从空白页开始

Coze 扣子编程虽然门槛低，但仍要求用户"配置工作流 + 选插件 + 理解触发器"——这对普通教师还是太难。XClaw 必须再下沉一层：

① 种子 Skill 包先行·教学专家定义 20 件普适教具（出题/批改/评语/学情周报…）—— 教师打开 XClaw 不是空白页，是 20 个能用的工具
② 教师只做"模板微调"·90% 教师停在 L1 模板微调（改参数）+ L2 自然语言生成（说一句话 AI 生成新 Skill）—— 不要求教师懂工作流
③ 算法配合做难的·教师数字分身（L4）让算法做底层风格建模，教师只标记"不像我"——不要求教师懂 prompt / 微调 / RAG
④ 教学语境深度封装·每个 Skill 都内置教学法（布鲁姆分类 / SOLO 分类 / 苏格拉底追问）—— 不是通用工具加一层教育皮肤

对 XClaw 战略选择的 3 条启示

1OpenClaw 内核选型再次得到印证

连字节都直接接 OpenClaw（不自己造），说明这是行业 runtime 标准。XClaw V0.2 已基于 OpenClaw 二开，方向无误。后续 2027 自研"高教龙虾"内核时也要保持与 OpenClaw 的 Skills 兼容——既不被锁死，又能跟得上行业生态。

2不抄"半技术 SaaS"路线 · 教师不是产品经理

Coze 扣子编程的"业务装配"路线对普通教师还是太难（要懂工作流、要选插件、要配触发器）。XClaw 必须再下沉——从"教师配置 Skill" 退到 "教师在已有 Skill 上微调"。这是教学场景 vs 通用业务场景的本质差异。

3价格参照 · 本地化定价已有竞争力

Coze 扣子编程 49 元/月 SaaS（年 588 元 / 人）—— XClaw 本地化版 18 万/校/年，假设 1 校 200 教师，年人均 900 元。同价位但 XClaw 给的是"本地化部署 + 教学场景特化 + Skill 跨学期累积"，价值密度远高。销售话术可以直接说："我们不卖 SaaS 月费，卖的是学校的教学数字资产容器。"

一句话总结

Coze 扣子编程是镜子，照清 XClaw 的差异化

同样基于 OpenClaw 内核，扣子编程包装成 49 元/月 SaaS 给半技术用户做业务流程，XClaw 包装成本地化教学工作台给教师做备建课/教学/考评/督管。

产品形态不同、目标用户不同、计费模式不同、差异化定位不同——看清扣子编程，反而能确认 XClaw 在产品光谱上的正确位置：教学场景专用，教师友好，不要做"半技术 SaaS"。
