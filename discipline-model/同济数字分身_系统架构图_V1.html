系统架构与能力映射

本图呈现同济数字分身方案的端到端系统架构——从最底层的教师原始数据（论文/视频/讲义）， 经由 Coral 知识资产层、XClaw 智能体运行时（核心调度层）、AI 模型层， 最终在教师/学生/管理者的应用空间产生价值。方案采用 RAG + Prompt Engineering + 多租户隔离 （非模型微调），在 8 月底一期交付 3 位样板教师的完整数字分身能力。

DATA FLOW

核心数据流 · 三路

数字分身不是单向的"调用 API 返回答案"，而是三路数据流的长期协同：教师数据沉淀流 / 学生提问响应流 / 教师纠偏反向流。 三路都由 XClaw 统一调度。
FLOW · 01 · 一次性 + 增量

🔻 教师数据沉淀流

从数据源 → Coral 知识资产 → 向量索引，由 XClaw 触发并跟踪状态。

  1. 教师上传论文 / 讲义 / 视频 到个人空间 L1→L6
  2. XClaw 调度 T1 多模态采集入库 Skill
  3. 调度 T2 文档解析 + ASR 转写（并行）Skill
  4. 调用切片 + 向量化，写入教师命名空间 L3
  5. 教师画像引擎提取风格标签和常用表达 L5 新建
  6. 生成数字分身 Agent Meta并注册到 Skill 库 L5 新建
FLOW · 02 · 高频 · 毫秒级

🔄 学生提问响应流

学生提问 → XClaw 调度 → 数字分身回答 → 源溯源 → 学生看到。

  1. 学生在答疑界面向"王教授数字分身"提问 L6→L5
  2. XClaw 意图识别：判断是否要直接答还是苏格拉底引导 Skill
  3. 调用 /teacher-kb-retrieve：按王教授 namespace 检索 Skill
  4. 加载王教授的画像 Prompt + 上下文注入星火 X2 L4
  5. 调用 /socratic-dialogue：生成反问或引导答案 Skill
  6. 调用 /source-trace：逐句标注原始切片 Skill
  7. 响应流式回传给学生 · 同步渲染溯源卡片 L5→L6
FLOW · 03 · 低频 · 高价值

🔺 教师纠偏反向流

教师发现回答问题 → 纠偏 → 更新 Prompt 或知识库 → 下次回答改进。

  1. 教师在纠偏面板点✏️ 我来修正 L6→L5
  2. XClaw 调用 /correction-feedback Skill 新建
  3. 判断纠偏类型：Prompt 级（风格偏差）
    / 知识库级（事实错误）
  4. Prompt 级 → 更新 教师画像 Prompt 模板 L5
  5. 知识库级 → 写入 M3 问答标准库 供首选命中 L3
  6. 创建新版本号，打包纠偏记录 L5
  7. 下次学生提同类问题时，直接返回教师纠偏后的答案 · 闭环完成

COREXClaw 能力清单 · 10 项核心能力

本方案对 XClaw 的依赖点 · 哪些是现有复用、哪些需要扩展、哪些需要新建

# XClaw 能力 状态 在本方案中的作用 改造点
01
Agent 运行时调度引擎
Agent Runtime Scheduler
 REUSE 学生每次提问的首站。负责意图识别、Skill 选择、任务编排、响应合成。 直接复用 XClaw v1.0 已有能力
02
多智能体 & Agent Meta 管理
Multi-Agent Registry
 NEW 每位教师注册为一个独立 Agent 实例，持有专属 Prompt、命名空间、风格元数据、纠偏版本。 XClaw 当前 Agent 概念需要扩展为"按用户多实例 + Meta 隔离"
03
教师画像引擎
Profile Extraction Engine
 NEW 从论文 / 讲义 / 视频转写中自动抽取教学风格标签、常用表达、解题路径，作为 Prompt 个性化的元数据来源。 新建 Skill + 数据管道
04
Skill 库与编排
Skill Library & Orchestration
 EXT 本方案新增 6 个 Skill、复用 4 个，全部挂载到 XClaw Skill 库统一管理。 XClaw Skill 注册机制已有，本方案扩展 10 个具体 Skill
05
RAG 检索路由
RAG Retrieval Router
 EXT 按教师命名空间向 Coral 发起检索，融合三路召回（向量 + 关键词 + 图谱）。 Coral E1 基础检索有，需增加多租户隔离和 namespace 路由
06
上下文管理
Multi-level Context Management
 NEW 会话级（当前对话）/ 学生级（这个学生的历史）/ 课程级（这门课的公共上下文）三层上下文传递。 全新建设
07
溯源链路追踪
Provenance Trace
 EXT 记录每次回答的"召回切片 → 重排位次 → LLM 引用 → 渲染句子"完整链路，供源溯源卡片使用。 Coral E3 底层能力有，需封装为 XClaw 可用 API
08
纠偏反馈回路
Correction Feedback Loop
 NEW 教师纠偏动作落库 · 区分 Prompt 级 vs 知识库级 · 反向注入 · 版本管理 · 下次同类问题直接返回修正答案。 全新建设 · 是"教师负责制"的技术体现
09
多租户隔离
Multi-Tenant Isolation
 EXT 每位教师的知识库、Prompt、纠偏版本、上下文彼此隔离，互不干扰。是"一人一空间"的技术底座。 XClaw 已有简单隔离，本方案深化为教师级 namespace
10
MCP 集成接口
MCP Integration Layer
 EXT 对接 Canvas / 小雅教务 / 星火 X2 / ASR / Embedding 的标准化协议，实现"融合共生"。 XClaw 已有 MCP 骨架，需实现 Canvas + 小雅的具体 connector

SKILLS本方案调用的 10 个 Skill

6 个新建 · 4 个现有复用 · 全部挂载到 XClaw 统一 Skill 库

/teacher-profile-build NEW

教师画像构建

从论文 / 讲义 / 视频转写中提取教学风格、常用表达、学科术语，生成个性化 Prompt 元数据。一次性 Skill，教师入驻时运行一次。

/teacher-kb-retrieve NEW

教师私有知识库检索

按教师命名空间发起 RAG 召回，融合向量 + 关键词 + 图谱三路检索。高频 Skill，每次学生提问调用一次。

/socratic-dialogue NEW

苏格拉底式对话

判断问题类型 · 决定是直接给答案还是反问引导 · 生成反问 · 控制轮次。本方案最有技术含量的 Skill。

/source-trace NEW

逐句源溯源

在 LLM 生成后，对答案中每一句做"原始切片反向定位"，生成可渲染的溯源卡片数据（视频位置 / 论文 DOI / 教材页码）。

/correction-feedback NEW

教师纠偏反馈

接收教师的👍/✏️/📚 动作，自动分类为 Prompt 级或知识库级纠偏，写入对应存储，生成新版本号。

/answer-hotmap NEW

答疑热力分析

统计一段时间内"哪些问题被重复提问"、"哪些学生答疑最多"，推送给教师做线下指导——减负而非监控。

/multimodal-ingest REUSE

多模态采集（Coral T1）

直接复用 Coral 的 T1 多模态采集工具——支持音视频图片解析、小程序拍照上传、微信公众号导入。

/doc-parse REUSE

文档解析（Coral T2）

复用 Coral T2 文档解析。PDF/Word 效果 OK，PPT 解析本项目专项优化（见 A4 风险）。

/course-report REUSE

课程报告生成

基于 XClaw 已有的"/course-summary" Skill 扩展：每节课后生成内容覆盖 / 互动参与 / 知识点分布报告。

/faq-match REUSE

FAQ 首选命中（Coral M3）

学生问题先在教师审核过的问答标准库中匹配，命中则直接返回——这是回答质量的"安全垫"。
