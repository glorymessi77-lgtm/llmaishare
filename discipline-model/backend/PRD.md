# Coral 小雅学科大脑 · 后台层 PRD

> 版本：v1.0 · 更新时间：2026-04-15 · 负责人：Coral 产品团队

---

## 0. 产品总览

Coral 小雅学科大脑是面向高等教育院校的学科级智能化教学平台，整体架构分为：前台层、**后台层**、工具层、引擎层、外部对接层、基础设施层。本 PRD 聚焦「后台层 Backend」，定义面向学校管理员、学科主任、教研组长的五大核心管理模块。

| 模块名 | 一句话价值 |
|------|----------|
| 权限与安全管理 | 让平台的每一次操作都可追溯、每一项权限都有依据 |
| 智能体配置 | 把"智能体"变成可配置、可版本化、可精准下发的标准产品 |
| 知识共建质控 | 以"教师共建、三级质控"方式沉淀高质量校本知识库 |
| 督导与运营看板 | 让校领导与学科主任一眼看清平台教学效能与风险 |
| 术语管理后台 | 用统一的学科术语为上层所有智能体提供语言基石 |

---

## 1. 权限与安全管理

### 模块概述
承担整个平台的"身份、权限、审批、审计"四大安全职责。通过 RBAC + 校级审批流的组合，保障一所高校内多个学院、多个角色、多层级审批路径的落地能力。

### 目标用户与场景
- **系统管理员**：教育信息化中心负责人，日常进行角色维护、审计抽查
- **校级管理员**：校长 / 副校长 / 教务处长，审批跨学院权限申请
- **学科主任**：处理本专业授课教师的日常权限申请
- **被管理对象**：授课教师、助教、学生（被分配权限者）

### 功能清单
1. **角色管理**
   1.1 5 级内置角色（系统管理员/校级管理员/学科主任/教师/学生）不可删除
   1.2 支持「自定义角色」，可继承某内置角色的权限并增减
   1.3 角色下成员列表维护（批量导入/转移/离职处理）
2. **权限矩阵**
   2.1 按模块分组展示所有权限点（查看/新增/编辑/删除/审核 五种原子权限）
   2.2 按角色进行可视化勾选，支持一键重置为默认
   2.3 变更前 diff 预览，保存后同步所有在线用户
3. **校级审批流程**
   3.1 流程可视化配置（最多 5 级节点，支持并签/或签）
   3.2 待审批工作台，支持「通过 / 驳回 / 打回修改」
   3.3 审批通知：站内信 + 微信模板消息（通过外部对接层）
4. **操作日志审计**
   4.1 所有写操作全量留痕（操作人/角色/模块/对象/IP/结果/时间）
   4.2 按模块、角色、时间范围多维筛选
   4.3 日志保留 3 年，支持 CSV 导出

### 关键流程（文字版）
以"教师申请成为某学科主任"为例：教师在前台发起申请 → 学科当前主任收到"待学科审核"任务 → 通过后流转到校级管理员"待校审" → 校长通过后系统自动写入新角色关系并下发权限 → 审计日志记录四个节点的操作人与时间。

### 核心页面
- 角色与权限页：左列角色列表 + 右侧权限矩阵表格
- 校级审批页：流程图（4 步可视化）+ 待办列表（按行操作）
- 操作日志页：全量表格 + 筛选器

### 数据字段建议
- Role: role_id / name / inherit_from / subjects[] / description / is_system
- Permission: perm_code / module / action / description
- ApprovalTask: task_id / applicant / target / current_node / status / history[]
- AuditLog: log_id / actor / role / module / action / target / ip / result / ts

### 权限与边界
- 系统管理员拥有全部权限，但对系统管理员角色的任何修改需二次确认
- 所有审批事件均依赖本模块，不允许绕过审批的直接授权

### 协同关系
- 是 M9/M6/M2 所有操作的前置鉴权
- 审批结果通过事件总线广播给 M9（角色变更 → 智能体可见范围更新）
- 日志数据是"督导看板"风险预警的重要数据源

## 2. 智能体配置

### 模块概述
统一管理平台内全部学生端、教师端智能体的完整生命周期，包括基本信息、Prompt 版本、知识库挂载、下发范围。后台配置后由工具层/引擎层消费，前台层调用。

### 目标用户与场景
- **系统管理员**：搭建新智能体、调优全局参数
- **学科主任 / 课程负责人**：在被授权范围内微调 Prompt、挂载学科知识库
- **授课教师 / 助教**：不能修改配置，但可以在沙箱中调试并提交 Prompt 优化建议

### 功能清单
1. **智能体列表**
   1.1 按使用对象分组（学生端 / 教师端）
   1.2 每项展示：图标、名称、描述、版本号、下发状态、日活
   1.3 支持新建、复制、归档、搜索
2. **基本信息**
   2.1 智能体 ID、适用学科、使用对象、引擎模型选择（学科引擎-V3 等）
   2.2 最大上下文、温度、对话轮次上限、超时时间
   2.3 安全策略：内容过滤级别、敏感话题阻断规则、外链限制
3. **Prompt 管理**
   3.1 分区编辑：System Prompt / Few-shot 示例 / 兜底回复
   3.2 版本列表（创建草稿、发布、回滚、版本对比）
   3.3 沙箱调试：在独立环境试跑，不影响线上
4. **知识库挂载**
   4.1 多知识库挂载，支持权重调整、启用/禁用、灰度
   4.2 展示每个知识库的条数、最近更新时间
5. **下发范围**
   5.1 学院/专业/年级/行政班树形结构勾选，支持全选/反选
   5.2 灰度下发（按比例或指定教学班）
   5.3 下发前校验：引擎容量、Prompt 合规、审批状态

### 关键流程
学科主任在基本信息/Prompt 中完成配置 → 复制为新版本 → 沙箱调试通过 → 提交发布（触发 M5 审批）→ 审批通过后选择下发范围 → 点击"下发"，系统推送至对应教学班的前台智能体。

### 核心页面
- 左侧列表（300px）+ 右侧详情（基本信息/Prompt/知识库/下发范围 四 Tab）
- 版本选择下拉 + 版本对比浮层
- 下发确认 Modal（预览范围 + 影响用户数）

### 数据字段建议
Agent: agent_id / name / audience / subjects / model / params / status
PromptVersion: version_id / agent_id / system / few_shot / fallback / author / published_at
Deployment: deploy_id / agent_id / version_id / scope (college[] / major[] / grade[] / class[]) / percent / ts

### 权限与边界
- 下发权限需经 M5 审批
- Prompt 修改需进入"待审核"状态，不允许直接覆盖线上

### 协同关系
- 消费 M2 术语、M6 知识库
- 配置结果被引擎层调用，运行指标回流至督导看板

## 3. 知识共建质控

### 模块概述
管理"教师共建 → 三级质控 → 入库 → 下发 M9"的完整知识生产链路，平衡速度与质量。同时提供贡献度激励与知识缺口洞察。

### 目标用户与场景
- **授课教师 / 助教**：贡献知识卡 / 例题 / 对比图
- **学科主任 / 课程负责人**：完成学科级学术审核
- **校级管理员**：完成合规审核与最终入库
- **系统管理员**：配置质控规则、查看全局报表

### 功能清单
1. **质控概览**
   1.1 4 项 KPI（待审/今日通过/驳回率/贡献者数）
   1.2 三级流程图（机审 → 学科审 → 校审），节点显示数量
2. **待审知识**
   2.1 表格列：标题/学科章节/贡献者/类型/机审评分/提交时间/状态
   2.2 点击行展开侧抽屉，查看完整内容、关联术语、机审详情
   2.3 操作：通过 / 驳回 / 打回修改（需填写意见）
3. **机审规则**
   3.1 查重（Simhash + 向量检索）
   3.2 术语合规（基于 M2 校验专业术语表达）
   3.3 格式规范（字段齐全、字符范围）
4. **贡献度仪表盘**
   4.1 月度排行榜（按入库数、加权通过率）
   4.2 个人贡献详情（历史、驳回原因统计）
5. **知识缺口发现**
   5.1 基于前台学生提问聚类 + 命中率分析
   5.2 可视化标签云（热度分级）
   5.3 一键生成"知识补齐任务"派发到课程负责人

### 关键流程
教师在前台提交 → 机审自动评分并打标（低于 60 分直接退回）→ 进入学科主任队列 → 通过后进入校审 → 校审通过后自动入库，并触发 M9 对应智能体的知识库版本更新。

### 核心页面
- KPI 栏 + 流程图 + 待审列表 + 贡献度 + 缺口标签云
- 知识详情抽屉（520px 右侧，含三按钮：打回/驳回/通过）

### 数据字段建议
Knowledge: kid / title / subject / chapter / type / body / contributor / status / machine_score / versions[]
QcRecord: qr_id / kid / node / reviewer / action / comment / ts
Gap: gap_id / topic / hot_score / related_questions[]

### 权限与边界
- 机审无法覆盖学科主任判断，即使高分也必须人工通过
- 校审只可通过/驳回，不可反向修改内容

### 协同关系
- 输出驱动 M9 知识库版本
- 贡献度数据可用于 M5 角色晋升参考
- 缺口数据回馈至督导看板

## 4. 督导与运营看板

### 模块概述
面向校领导与学科主任的「看学业、看系统、看风险」全景数据台，沉淀日常督导工作所需指标、可视化与预警。

### 目标用户与场景
- **校长 / 教学副校长 / 教务处长**：决策学期重点、评估投入产出
- **学科主任 / 课程负责人**：定位本专业弱班、弱点
- **系统管理员**：监控平台健康度

### 功能清单
1. **KPI 全景**：DAU、答疑数、达成率、BadCase 数、术语覆盖率、学期更新数 6 项核心
2. **趋势分析**：学科达成度、答疑成功率、BadCase 占比多线趋势
3. **学科 × 年级热力表**：快速定位弱班（覆盖大一至研二）
4. **BadCase 管理**
   4.1 类型筛选（回答错误/答非所问/合规风险）
   4.2 点击进入处理流（关联 Prompt 版本回溯）
5. **预警中心**
   5.1 自动触发规则（达成率连续下滑、BadCase 超阈值、术语版本过期等）
   5.2 支持标记已处理、指派责任人、关联工单

### 关键流程
系统按日/周定时计算各项指标 → 触发规则命中时生成预警 → 推送给对应责任人 → 管理员在看板点击预警卡片进入详情 → 处理完成后标记关闭，日志保留。

### 核心页面
- 顶部时间筛选（今日/7 日/本月/学期）
- 6 KPI 卡 + 趋势折线图 + BadCase 列表 + 预警列表 + 热力表

### 数据字段建议
Metric: metric_code / value / dimension / period / ts
BadCase: bc_id / agent_id / version_id / question / answer / category / severity / status
Alert: alert_id / rule_id / level / target / status / assignee / ts

### 权限与边界
- 校级管理员可见全校数据；学科主任仅可见本学科
- 数据看板只读，所有处置动作跳转至原模块

### 协同关系
- 聚合 M5 日志、M6 质控、M9 运行、M2 版本数据
- 预警规则配置由系统管理员维护

## 5. 术语管理后台

### 模块概述
学科术语是 Coral 所有智能体回答的"语言地基"。本模块负责术语的新增、审核、版本发布与历史追溯，确保全平台同一概念的表述一致。

### 目标用户与场景
- **学科主任 / 课程负责人**：主导本学科术语的持续维护
- **授课教师 / 助教**：提交新术语或修订建议
- **系统管理员**：负责版本发布节奏与全局治理

### 功能清单
1. **学科导航**
   1.1 左侧树结构（8 大学科门类 + 二级方向）
   1.2 每项显示术语数量
2. **术语词条**
   2.1 表格列：术语/释义/状态/版本/更新时间
   2.2 状态：已发布 / 待审核 / 已废弃
   2.3 单条操作：编辑 / 审核 / 历史版本
3. **新增与编辑**
   3.1 必填：名称、学科、编码（自动生成）、释义
   3.2 可选：同义词、引用教材
4. **批量导入**
   4.1 支持 Excel / CSV，提供模板
   4.2 导入后进入批量审核队列
5. **审核流**
   5.1 通过 / 驳回 / 打回修改
   5.2 审核意见必填（驳回/打回时）
6. **版本管理**
   6.1 版本列表（构建中/当前生效/已归档）
   6.2 版本对比（diff 视图）
   6.3 发布新版本（需系统管理员二次确认）

### 关键流程
教师提交新术语 → 进入审核队列 → 学科主任审核通过后标记"纳入 v2026.04 草稿" → 到学期节点由系统管理员点击"发布新版本"→ 下发至知识库、M9 智能体、M6 机审规则。

### 核心页面
- 三 Tab：术语词条 / 审核队列 / 版本管理
- 新增/审核/导入 3 个 Modal

### 数据字段建议
Term: term_id / code / name / subject / definition / status / version / aliases[] / updated_at
TermVersion: version_id / name / published_at / change_stats / snapshot_ref
TermAudit: audit_id / term_id / change_type / submitter / reviewer / comment / ts

### 权限与边界
- 术语编码一经发布不可修改
- 废弃术语不物理删除，仅改状态，保留历史查询

### 协同关系
- M6 机审依赖术语库进行合规校验
- M9 Prompt / 知识库在生成回答时优先引用最新版本术语
- 版本变更事件推送至督导看板"学期更新数"指标
