CHAPTER 01

从学校的处境说起

讲清楚学科大模型,先看学校面前是个什么处境、有什么资产、要的是什么。用"学校怎么想"反推"我们怎么做"。

STEP 01 · 处境
通用模型不懂"这个学科"
通用大模型在具体学科上回答不准、不专业、没出处。食品学科问题、医学诊断、法学条文、历史考据……通用模型只能给"大概对的答案",学校需要"专业级的答案"。
STEP 02 · 资产
学校有数据,也有场景
教材、讲义、试题、案例、论文、政策、标准、文献——学校自己有完整学科资产,场景也明确(学科建设、学科研究、学科应用)。问题不在缺数据,在怎么把这些数据用起来。
STEP 03 · 要的
要做出标志性学科成果
学校真正要的不是"装个 AI",是做出标志性成果——可以演示、可以传播、可以上国家智慧教育平台、可以支撑评奖评优。能持续做出标志性成果的学科能力,才是学校愿意投入的。

数据怎么进来 · 两种处理方式

结构清晰 · 拿来即用

教材 / 讲义 / 标准答案 / 政策文件

结构化的文本资料,扔进学科大脑就能用。系统自动完成解析、切分、向量化、入库。

需要清洗 · 我们的服务

扫描件 / 图片 / 音视频 / 口语化记录

这类资料需要先清洗、标注、结构化——三引擎 OCR、ASR 转写、学科 know-how 萃取,这是我们提供的陪跑服务。

CHAPTER 02

学科大模型 是什么

一句话定义
学科大模型,是基于学校自己的学科数据,把通用大模型、学科大脑、智能体创编平台融为一体, 让 AI 在具体学科上的专业表现显著优于通用模型,并支撑学校持续做出标志性学科成果的一套学科级 AI 解决方案。

不是模型更强,是数据用得更好;不是新发明,是把三件东西拼到一起——价值不在"有这三样",在怎么把它们组合好。

学科大脑 三驱动 · 让 AI 真正懂学科

DRIVER 01

检索增强

模型回答前,先去学科大脑调出学校资料——等于开卷答题。5 路混合检索(向量 + 全文 + FAQ + 图谱 + 摘要),答案可溯源到教材页码段落。

DRIVER 02

图谱约束

用学科知识图谱约束推理路径,按这个学科的概念和关系来想问题。融合向量库、全文检索、知识图谱三类底层引擎。

DRIVER 03

评测闭环

建一套学科评测集,自动化 + 人工双轨打分,答错的针对性优化。能力维度:知识覆盖准确度 / 逻辑推理 / 回答合规 / 引用溯源 / 表达质量。

都不靠改模型本身。主路径就是不训练模型,靠把学校数据用好——这是学科大模型的核心方法论。

七大模块 · 体系化保障

三驱动是 AI 回答时的运行机制,但它跑得起来,靠的是学科大脑七大模块的完整体系做底盘——每个模块都是三驱动的支撑:

① 知识接入 → 有完整数据,检索增强才有源头
② 知识加工 → 数据结构化,图谱约束才有路径
③ 质量治理 → 数据干净,运行机制不被脏数据污染
④ 共享共建 → 数据持续丰富,模型懂得越来越多
⑤ 知识应用 → 三驱动的执行场
⑥ 运营迭代 → 评测闭环的持续进化引擎
⑦ L0 技术底座 → 支撑三驱动的运行环境(国产化算力 / 推理 / 检索 / 存储 / 安全)

体系是底盘,机制是发动机,缺一不可。下面展开七大模块的具体能力。

CHAPTER 03

学科大脑 · 7 大能力模块

学科大脑是学科大模型的知识中台,覆盖从"原始资料进入"到"知识应用产出"的完整链路。7 大模块按能力流转顺序展开:知识接入 → 知识加工 → 质量与治理 → 共享共建 → 知识应用 → 运营与迭代 → L0 技术底座。
①
知识接入
定位:把师生的各类教学科研资料便捷汇入知识库,资料拿来即用。是学科大脑能力链的入口。

核心功能

  • 多格式文件接入:15 种主流格式——文档(PDF/Word/PPT/Excel/TXT/MD)、表格、图片、音视频,以及 FAQ 问答对专用接入
  • 多渠道接入方式:本地直传 / 对象存储 / 网页链接抓取 / 批量导入 4 种方式
  • 多模态采集与解析:扫描件、图片型 PDF 三引擎 OCR(标准 / 多模态视觉 / 本地化);课堂录音、讲座视频 ASR 转写
  • 学科原生接入增强:教材、教案、习题专用解析,自动识别章节、标记知识点
  • 批量与初始化:知识库批量创建、分类批量创建、资源批量上传

关键能力

  • 15+文档格式
  • 4 种接入渠道
  • 3 引擎OCR 识别
  • ≤ 20MB单文件
②
知识加工
定位:把原始资料自动加工成可被 AI 直接调用的学科知识资产,无需人工干预即可形成结构化知识。

核心功能

  • 知识库自动构建:文件转换 → 文档分片 → 向量化 → 入库 全流程自动化,实时追踪构建状态
  • 文档分片(8 种模式):智能拆分 / 极速 OCR / 深度 OCR / 智能识别目录 / 按长度 / 按目录 / 按分隔符 / 按行
  • 知识抽取:自动提炼 4 类——学科术语 / 问答对 / 知识图谱节点 / 内容摘要
  • 采编结果人工干预:切分、抽取结果可视化管理,教师可对结果人工校正
  • 学科级语义解析:理科公式标准化 / 化学反应式规范 / 历史时序梳理

关键能力

  • 8 种分片模式
  • 4 类自动抽取
  • 5 种分片参数
  • 100%流程自动
③
质量与治理
定位:保障学科知识库内容干净、不冗余、不矛盾、合规、可追溯;让学科应答始终基于可信源头。

核心功能

  • 问答对治理:FAQ 自动抽取问答对,人工审核 + 编辑,提升问答命中率

CHAPTER 05

学科大模型 · 用在哪里

学科大模型不仅服务于教学,更核心的是学科级专业能力。面向学科建设、学科研究、学科应用三个方向,在每个学科里都能落到具体的学科级专业问答。
