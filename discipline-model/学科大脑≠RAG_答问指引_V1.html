学科大脑 ≠ RAG
答问指引

创作者 · 孙玉欣 版本 · 内部话术 V1 发布 · 2026-04-15

RAG 的定义边界

RAG 只做三件事

向量检索
→
提示词拼接
→
大模型生成
RAG 在此范围外不进行模型调整、数据管理、结构化处理、校验或评估。它是一个技术模式，不是一套产品体系。

学科大脑的九大工程体系

01
语料工程
清洗 / 分段 / 标注 / 去重
02
知识图谱
多跳推理与实体关系
03 · RAG 在这
多路检索
RAG 只是其中一路（向量检索）
04
领域微调
SFT 与针对性微调
05
领域原子工具
化学 / 法条 / 公式计算等
06
Hook 校验层
越界 / 引用 / 数值 / 溯源校验
07
上下文与记忆
对话 / 会话快照 / 权限
08
评估体系
准确率 / 覆盖率 / 溯源率
09
Harness 编排
整合前八层的调度引擎
85%
RAG 大约只占学科大脑整体工程量的 10-15%。
另外 85% 是 RAG 不覆盖的学科化工程——语料工程、知识图谱、领域微调、校验层、评估体系、编排引擎。

推荐应答话术 · 三段式

客户问"你们和 RAG 有什么区别"时，按以下三段回答：

1
承认 RAG 是系统组件
不否认、不回避。先认可 RAG 的价值，表明我们的系统中确实包含 RAG 能力。
"对，学科大脑里确实有 RAG——向量检索是我们多路检索中的一路。"
2
划清 RAG 的技术边界
明确 RAG 只做"检索+拼接+生成"三步，不覆盖数据加工、模型调优、校验评估、调度编排。
"但 RAG 只管'检索到相关片段 → 塞给大模型'这一步，它不管数据怎么清洗、图谱怎么建、答案怎么校验。"
3
列举其他八项差异化工程
逐项说明 RAG 之外学科大脑做了什么——语料工程、知识图谱、领域微调、原子工具、Hook 校验、记忆系统、评估体系、Harness 编排。
"学科大脑还有语料工程、知识图谱、领域微调、溯源校验、评估体系、编排引擎……RAG 只占整体的 10-15%，剩下 85% 才是我们的核心壁垒。"
🚗
一句话类比
"说学科大脑是 RAG，就像说汽车是发动机。发动机很重要，但汽车还需要底盘、变速箱、刹车系统、安全气囊、车载电脑……RAG 是发动机，学科大脑是整台车。"
