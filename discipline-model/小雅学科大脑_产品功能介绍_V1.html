数据存储层
Storage Layer · Five-tier Architecture
学科大脑的数据底座由五种不同形态的存储协同组成：对象存储放原始资产， 关系库放结构化元数据，向量库支撑语义检索，图数据库承载图谱推理，缓存层吸收高频访问。 每一层都为上层能力服务，不可替代。
GROUP · 4.1

五级存储

05 tiers · 05 screens
数据存储层是学科大脑中完成度最高的一层——五种存储形态全部 ✓ 已具备。 一次典型的问答请求依次触达：缓存层先命中高频问题 → 未命中走向量库语义召回 → 补充图数据库关联推理 → 通过关系库拿元数据 → 从对象存储取原文定位。
D1 · OBJECT STORE
原始资产 · 对象/文档存储

承载 PDF / Word / 图片 / 音视频等原始文件，按学科、版本、来源分桶；支持加密、快照与生命周期管理。

D2 · RDBMS
元数据与业务 · 关系数据库

资产元数据、账号权限、版本记录、任务状态等结构化数据，支撑管理后台与审计功能。

D3 · VECTOR DB
语义检索 · 向量数据库

存储切片的 Embedding 向量，支持大规模 ANN 检索；与 E1 混合检索引擎无缝联动。

D4 · GRAPH DB
关系推理 · 图数据库

承载知识图谱的节点与关系，支持多跳遍历、路径查询与图算法，为图谱增强检索提供算力。

D5 · CACHE
高频 FAQ · 缓存层

把高频命中的问答对直接缓存在内存层，常见问题毫秒级返回，显著降低后端检索压力与模型调用成本。

安全防护 & 全私有化部署
Security & Fully On-Premise Deployment
学科大脑面向 B 端高校与科研机构，数据不出校园是底线。 通过五层安全防护（物理部署 + 身份认证 + 权限控制 + 数据安全 + 内容安全）+ 全私有化部署架构， 保证从硬件到模型到业务的每一环都可控、可审、可隔离。
GROUP · 6.1

五层安全防护体系

4 of 5 layers ready · 02 screens
五层中仅第 1 层"物理部署安全"由学校本地化保障（机房、硬件、物理隔离），其余 4 层全部已具备： 身份认证层（OAuth2 / JWT / API 网关 / CSRF）、 权限控制层（三级角色 + 三层权限模型 + 租户隔离 + 权限守卫拦截）、 数据安全层（HTTPS + HMAC 签名 + 阿里云 OSS 11 个 9 持久性 + Nacos 配置加密）、 内容安全层（敏感词检测 + 分角色审核 + 输入输出双向过滤 + 动态策略管控）。

S · 权限控制层（示例）

Authorization · Role-Based Access Control

基于责任链模式的统一认证 + EDUADMIN / TEACHER / STUDENT 三级角色 + 个人 / 团队 / 查询三层权限模型。所有数据操作强制校验 schoolId 实现 租户级隔离，A 学校用户无法访问 B 学校任何数据。 KnowledgeBasePermissionGuard 统一拦截读 / 写 / Owner 三级访问。 所有敏感操作全链路审计留痕。

GROUP · 6.2

全私有化部署

03 deployment modes
全私有化部署架构已具备。学科大脑支持按学校诉求提供部署方案，数据与模型权重均不出校。 以下三档是常见交付形态，具体组合按项目落地。
🏛️
Mode A · 全栈私有化

硬件 + 存储 + 模型 + 业务应用全部交付到学校本地数据中心。支持信创国产化生态， 满足最高等级保密要求，适合重点高校与科研院所。

🏢
Mode B · 校园专有云

硬件由学校自建，软件由小雅交付；上层应用可按学院多租户切分。适合有机房、有运维 但不想自己做算法的高校。

☁️
Mode C · 专有 SaaS

由讯飞在学校独占集群内托管，合同期内数据隔离、专人运维。 适合中小高校与短期试点项目，部署最快。
