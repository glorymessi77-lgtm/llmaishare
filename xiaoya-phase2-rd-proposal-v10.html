小雅 3.0 二期产研立项

主线一 · 先补短板 围绕 3.0 已规划、830 前应上线而未完成的内容，按模块补齐；每项说清"现在做到什么程度、本期做什么、不做有什么影响"。
主线二 · 再定方向 战略收敛为四大方向：方向一是平台底座本身（超级智能体穿起全平台），方向二/三/四是解决方案，产品在基线产品上做好对方案的场景与竞争力支撑。
统一叙事：现状（做到什么程度）→ 本期开发要点 → 解决痛点 → 用户价值；每个方向末尾点到"未来发力方向"，不展开。知识库能力在开发要点中重点体现。
一

先补短板

3.0 想做还未完成的 6 大模块（830 前收尾为目标）
聚焦"已规划未上线 + 迁移未完成 + 客户共性"的补齐。凡能对下面四大解决方案形成竞争力的新特性，一律上移到主线二打造，本处只做平台自身短板补齐。
①合规安全P0 · 致命 · 集团硬性
现状
集团已出台产品上架统一要求，当前内容安全审核、安全渗透、AI 标识等尚未按新规完成，不达标即无法上架。
开发要点
UGC/PGC 内容安全审核；人工安全渗透缺陷修复；AI 标识；满足集团上架统一要求。
解决痛点
这不是用户痛点，而是平台无法合规上架的硬性卡点。
用户价值
满足监管合规硬性要求，保障平台安全上线、用户内容与数据安全。
②交付模式验证与本地化部署P0 · 产线共性需求
现状
规模化交付所需的第三方教务系统对接、混合式本地化部署跑通验证，需产线推进 POC 验证（POC）。
开发要点
第三方教务系统对接、SSO / OAuth2.0 / 企业微信移动端免登录；混合式部署与本地化方案验证；小雅 2.0 及智课 2.0 数据迁移；多语言扩东盟语种（英语/泰语 / 越南语）。
解决痛点
不完成验证，未来面临规模化交付无法落地的问题。
用户价值
打通自有平台集成、数据本地化与合规能力，支撑平台规模化交付。
③MOOC & SPOC 双场景统一（含数据特性补齐）P1 · 核心补齐
现状
平台本就支持 MOOC / SPOC 两种课程类型。到 7月15日 已基本实现功能层面一致性，仍有部分资源类型、关键特性（如 SPOC 自适应练习）未对齐；MOOC 运行（学习数据统计、统一门户后台、分学期运行）、自定义图谱迁移、五类画像报告迁移等尚未完成。
开发要点
围绕"教与学的流程统一、功能统一、数据统一"三个维度，从两个实际目的补齐：
① 关键特性两边对齐：两种课程类型的资源类型互通复用、共享课程包作业/讨论/题目下沉、SPOC 自适应练习等特性对齐；
② 历史客户迁移不受损：自定义图谱类型承接小雅 2.0 标签图谱、五类画像报告（教师/学生/课程/资源/院系）迁移、慕课运行数据统计与统一门户后台。
解决痛点
两种课程类型体验/资源不一致、历史客户迁移受损、教师重复建课、管理缺数据抓手。
用户价值
教师一次建课多场景复用、历史客户平滑迁移不丢数据、学生体验一致、管理有数据可用。
④AI 能力接入与智能化升级（平台自身体验补齐）P1 · 体验补齐
现状
AI 工作台与 AI 助学多入口割裂（已评审重新划分）；部分原有产品的 AI 能力尚未复用（"AI 为我讲解"、视频截图生成课件等）；AI 问数待升级。（任务团子已归入方向二未来课堂·课前备课。）
开发要点
AI 工作台智能体分类优化、AI 助学入口合并与重新划分；复用原有产品 AI 能力；升级 AI 问数（对话式数据分析）；支撑 Skill 模块化配置。
解决痛点
AI 入口割裂、体验不一致、已有能力未复用——都是"平台自己的问题还没解决"这一类。
用户价值
AI 入口统一、体验一致，降低使用门槛。
边界：更高阶、能随方案售卖的新特性——AI 建课（能力/问题图谱智能生成、资源智能打标签）、任务团子生成、附件批改等——一律上移到方向二/三/四打造，本处只做平台自身体验补齐。
⑤移动端P1 · 正式上架
现状
安卓已上架应用商店；iOS 账号注销功能已完成、正在提交审核。目标是把小雅 2.0 移动端功能迁移到 3.0——因主体变更需以新主体重新上架。
开发要点
新主体重新上架与应用商店审核；研发适配（SPOC/慕课章节资源混排、自主注册与登录逻辑）；消息推送（依赖证书配置）；企业微信移动端免登唤起。
解决痛点
移动端功能未迁移齐、触达能力缺失。
用户价值
师生随时随地移动教学与学习，提升平台活跃与触达。
⑥教务与课程管理增强P1 · 迁移完善
现状
多为未迁移过来的功能完善，非新需求：平台级跨校课程审核流程未做（当前仅本校）、教师自主开课转教务课程（承接 2.0）、课堂提问/定位签到（2.5 需求）、教师待办（承接 2.0 模块）、教师预警。
开发要点
跨校课程审核流程闭环（平台级）；教师自主开课转教务课程；课堂提问、定位签到、自定义标签管理；教师空间新增教师待办、教师预警。
解决痛点
教务管理不规范、功能未迁移齐、教师管理负担重。
用户价值
教务管理更规范高效、教学互动与考勤更便捷、减轻教师负担。
二

再定方向

四大方向：方向一为平台底座，方向二/三/四为解决方案
战略收敛为四个方向：智慧教学一体化平台（整体底座） —— 支撑未来课堂、支撑未来学习中心、支撑学科大模型。

方向一不是解决方案，是平台本身：把平台做好、用超级智能体把散落的场景与功能穿起来，是各方向的公共底座——凡能做到基线、各方向都能用的能力，都归方向一。方向二/三/四是解决方案（产品）：小雅在其中扮演核心角色，我们要想清楚每个方向做哪些场景与能力，让客户在讲到核心功能时"眼前一亮"、形成竞争力。方向即战略，定了不轻易调整；产品在产品力上为方案备好"粮草"。

每个方向写法：先说 场景/智能体做什么，再把 知识库能力在开发要点里重点体现（知识库贯穿四个方向）。
本期（至 10 月）优先级共识
  1. 知识库对各场景的支撑与打通——本期核心，把每个场景用知识库支撑好、治理好。
  2. 超级智能体——圈定有限范围：调度已有智能体开发平台能力 + 少量高频调度场景，先跑通。
  3. 多智能体课堂等前沿能力——以解决方案 / 项目倒推，借鉴成熟架构（如清华），逐步推进、后置。
方向 1平台底座智慧教学一体化平台 · 超级智能体
为教师、学生、管理者提供一个"超级智能体"——以自然语言为入口，理解用户意图后一句话调度全平台（如 AI 助学 / AI 问数）、一句话生成智能体，把散落的平台拧成一个系统。
场景功能现状
已有一个基础入口（超级智能体浮窗），但平台散落的智能体、星火自研智能体尚未归集，仍需单入口逐个找。
知识库现状
已具备多类型知识库与统一/多库检索能力，但跨产品、全场景的一致调用尚未打通。
开发要点
本期圈定有限范围先跑通
① 调度（本方向核心）：一句话调度平台散落应用与页面（解决功能藏在三级路径）；调度小雅平台已有的智能体（PPT 生成、出题、批改等）——把成熟能力包装成 API → 写成 Skill 注册 → 供检索调度；统一对话式智能体（助教/助学）入口，按场景调度不同智能体；对话式问数。
② 生成智能体/工作流（接入星辰 Agent 一句话创编）：由超级智能体理解用户意图、下达指令，接入星辰 Agent 平台的"一句话创编 workflow 型智能体"能力（拼装平台已有的对话式 / workflow 智能体接口），实现一句话生成智能体 / 工作流；本期不做代码级的智能体生成。生成物可发布到智能体广场给学生使用。（面向学科项目的一句话生成应用展开见方向 4。）
③ 接口与差异化（与竞品不同）：以 MCP 方式把小雅教学数据接口开放给自定义智能体，使其嵌入真实教学场景——把平台能力封装为 MCP 服务工作量大，故本期先做少量学情接口的 MCP 试点验证可行性；全量接口封装、被外部平台调用等移未来逐步推进。
④ 三级知识库共享与跨产品互通：支持个人 / 团队 / 院校三级知识库共享、资源互通，小雅 / 学科大模型 / XClaw 跨产品互通引用，解决用户权限与使用路径上的卡点（北京物资学院需求）；支持教师将个人知识库发布为共享知识库并建立审核机制。
知识库能力（重点） 权限与知识库组织（多类型库） 统一检索（多库/跨库） 数据回流与评测（平台级度量）
解决痛点
功能藏在三级路径、逐层找；自研能力散落各平台没盘活；平台"散"而不"通"。
用户价值
一个超级智能体把平台拧成系统——一句话调度全平台 + 生成智能体/应用 + 数据可用，支撑规模化交付与差异化竞争力。
投入与代价（需拉通评估）：本期"生成"收敛为接入星辰 Agent 的一句话创编 workflow 型智能体（不做代码级生成）；MCP 收敛为少量学情接口试点（全量封装工作量大、移未来）。对标参照——对外看支付宝/微信（传统入口 + 超级智能体入口并行），对内看星光教师超级助手（重兵投入、当前仅纳管工作流智能体 + 生成 H5 交互）。本期圈定有限范围、按方案/项目推进，先讲清实现逻辑、成本与优先级。
未来发力：代码级智能体生成；MCP 全量接口封装与外部平台调用；逐步扩展调度场景、多智能体协同、AICoding 级生成。
知识库要建什么：课程知识库（教学班群组）并线〔P0，T610 在建可并入〕· 数据回流+评测〔P0〕· 统一库组织与调用口径 · 三级知识库共享与跨产品互通引用（含发布审核机制）。先把课程知识库统一并线 + 数据回流评测做成平台级底座。
方向 2解决方案未来课堂
上好一堂课的完整闭环——课前备课 · 课中上课 · 课后批改 · 教学评价，对齐产线"教学评管"方案，做成可对外宣、可交付。
为何落此方向：未来课堂是"师-生-智三元互动"的育人主阵地，需重制课前-课中-课后完整路径。其中作业批改是高校应用最频繁的 AI 刚需（应用次数超过其它智能应用总和），批改-订正-重交构成课后闭环，故本期优先落"备课+批改"这两个成熟刚需；课中多智能体课堂以开源框架小步试点，前沿能力不铺大摊子。
为何建此知识库：三元课堂要"实时核验、批改有依据"，不能凭空说话，必须挂课程知识库溯源——所以要一键建库 + 受控生成封装；不建则核验无凭、批改通用化、人机协同落不了地。
场景功能现状
备课（生成课件 / AI 互动课堂已有）、批改（部分已有）；课中三种教学模式已支持——讲授+互动（选人/随堂练习/AI 助教即时评价）、项目式教学 PBL、教学活动设计闭环（导入-前测-协作-后测-总结）；但课堂辩论/角色扮演/圆桌等多智能体课堂尚无自建基础，此前仅对接过清华的多智能体课堂，开源智能体框架待调研试点；教学评价（考试/督考巡课/课堂教学评价融合方案推进中）。
知识库现状
批改与核验需挂课程库溯源，但课程库多为空或未关联；基座检索、受控生成、溯源、兜底已具备，卡在"一键建库/自动关联"。
开发要点
本期先做备课/批改，课中互动次之
课前 · AI 建课备课：复用平台已有备课能力，依据知识库章节/课程目录做自动化备课；老师手头资料自动打标签；任务团子生成配套资源；章节下新增 AI 互动课堂（基础版）/ 生成课件。
课中 · 多智能体课堂（调研试点，收缩边界）：课堂辩论、角色扮演、圆桌讨论等由智能体参与的三元互动，此前仅对接过清华的多智能体课堂、尚无自建基础；本期先调研开源智能体框架并做小范围试点验证可行性，不追求大而全，跑通再逐步扩展。
课后 · 作业自动批改：支持图片、Word、文档附件的自动批改（默认自动、复用现有批改引擎），依据课程知识库难度做受控批改与判分依据。
教学评价：对齐产线"教学评管"方案（引用为依据），融合考试、督考巡课（复用资源中心直录播 + 巡课驾驶舱接入管理空间、督导流程沿用小雅现有）、课堂教学评价；与 VI 课堂行为分析融合。
知识库能力（重点） 检索召回（课程库检索） 受控生成 + 引用溯源 拒答兜底（课堂核验有据） 受控生成封装（批改判分依据）
解决痛点
课前中后割裂；批改通用化、不支持附件、无判分依据；课中三元互动缺协同、缺特色；评价未闭环。
用户价值
课前中后 + 评价形成完整闭环、有据可依，构成差异化竞争力。
未来发力：多智能体课堂规模化/深度化（借鉴成熟架构如清华，或软硬结合——大屏分屏选多智能体、讨论记录回流学习空间）；课堂教学实录 + AI 智能切片资源沉淀、进阶版 AI 互动课堂等重投入能力，视试点效果与市场洞察再评估推进。
知识库要建什么：一键建库/自动治理〔P0〕· 课堂核验接入课程库〔P1〕· 受控生成封装（批改依据）〔P2〕。
方向 3解决方案未来学习中心
让学生自己能学起来——自主学习问答、解题辅助、按知识点学习和精准定位。
场景功能现状
学生自主学习相关场景，产品侧多为 prompt 工程，解题/讲解与本课程关联弱。
知识库现状
目前仅 AI 助学接了 1 个知识库问答，其余学习场景基本未接库；基座双层检索、溯源已具备，缺图谱节点推荐、视频时间戳定位。
开发要点
本期核心：把每个学习场景用知识库支撑好
场景：自主学习问答、答题辅助（解题挂教材片段并溯源）、知识点学习（图谱节点推荐）、知识点视频定位（本期基于视频语音识别 ASR 的文字做切片溯源、可定位到对应时间段；做不到画面识别与对齐——如部分 PPT 画面难以对齐，画面级切片移未来规划）。每个场景以知识库检索+溯源为底，逐个接入、逐个支撑好。
知识库能力（重点） 检索召回（个人库+课程库双层） 受控生成（AI 生成学习视频/内容） 引用溯源（附来源可跳转） 图谱节点级推荐 多模态视频 ASR 文字切片溯源（画面级对齐移未来）
解决痛点
答无依据/超纲/幻觉、视频定位不准、无法按知识点个性化推荐。
用户价值
学生"答有依据、能跳讲解片段、按知识点推荐"，真正学得起来。
未来发力：视频画面级识别与对齐（PPT/板书画面切片）；专业全景中心闭环、个性化学习路径。
知识库要建什么：一键建库/自动治理〔P0〕· 图谱节点级推荐〔P1〕· 学生问答三合一接入〔P1〕· 视频 ASR 文字切片溯源〔P2〕（画面级对齐移未来）。
方向 4解决方案学科大模型
让学院的领导和老师，把自己的知识库建好、管好、用好——基于学院既有数据，越用越准；并支持一句话生成智能体/应用。
场景功能现状
具体到各学科的知识库目前基本未建、内容管理与优化能力弱；一句话生成智能体依托智能体开发平台。
知识库现状
知识库数据解析、检索、召回、溯源能力已具备；但缺少数据回流、RAG 评测闭环、一键治理，缺少能直接让老师、学生、管理者用起来的治理功能。
开发要点
自动化建库（打通课程资源入库处理流程）：教师扔资源进来，系统自动判别资源类型，可公开资源自动解析建库、按敏感度自动推荐建库 → 沉淀课程级知识库（解决"课程资源与知识库资源处理独立""试题等敏感资源不便入库"的问题）。
场景治理（一键治理）：教师用自然语言提问题（"搜不到题"/"答非所问"/"说的不靠谱"）→ 系统自动匹配治理策略 → 一键治理 → 教师测试采信 → 沉淀治理经验技能。
数据回流：教师采信后的优质问答对 → 回流知识库 → 知识库越用越准。
一句话生成智能体（探索，随方向 1 推进）：复用方向 1 超级智能体接入星辰 Agent 的一句话创编 workflow 型智能体能力，面向学科项目实现一句话创建智能体（几乎所有学科大模型项目都需要建智能体开发平台）；本期以建库/管库/治理为主，生成能力作探索。
组合：科研助手（组合方案、驻研问答场景）；专业方向可随院级方案纳入。
基础版：模型 + 学校已有知识库数据，解决基础专业性问题（低成本落地版本）。
知识库能力（重点） 知识图谱 · 能力图谱（OBE） 检索 + 强反幻觉 + 精确溯源 数据回流与评测（面向学科调优）
解决痛点
学科库为空、答不准、无法按学科持续迭代；收紧到"学院领导及教师可管可用"，避免"那么多学科怎么支撑得过来"的挑战。
用户价值
学院教师自主把学科知识库建起来、用起来、越用越准；基础版低成本落地。
未来发力：更多学科纵深、制度库/科研文献库沉淀。
知识库要建什么：一键治理与自动化建库编排〔P0〕· 数据回流 + RAG 评测〔P0〕· 图谱扩展：能力图谱产品化〔P1〕· 科研文献库/制度库沉淀〔P3〕。
